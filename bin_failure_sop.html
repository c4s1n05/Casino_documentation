SOP — BIN Failure Investigation & Resolution

Defines how to detect, investigate and resolve scenarios where a specific Bank Identification Number (BIN) range is experiencing elevated decline rates. Covers root cause identification, SoftSwiss escalation, rerouting options and player communication standards.

📅 May 2026
👤 Owner: Payments Team
🔗 Platform: SoftSwiss
⏱ Detection SLA: 30 minutes
🃏
What is a BIN?

A Bank Identification Number (BIN) is the first 6–8 digits of a payment card number. It identifies the issuing bank, card scheme (Visa, Mastercard), card type (credit, debit, prepaid) and country of issue. When a specific BIN range has a high decline rate, it typically indicates an issue with that specific issuing bank — not with our platform or SoftSwiss. However, SoftSwiss controls the routing and may be able to resolve it via acquirer switching or BIN-specific configuration.

03
Investigation Steps

Investigation must determine whether the BIN failure is caused by the issuing bank, the acquirer, SoftSwiss routing configuration, or a specific card type issue. This determines the correct resolution path.

#Investigation ActionToolWhat It Tells You
1Pull BIN-level decline report from SoftSwiss back office — filter to failing BIN prefix, last 60 minutesSoftSwiss PortalConfirms the BIN range affected, volume and decline codes
2Identify the issuing bank using a BIN lookup tool — confirm bank name, country and card type (credit/debit/prepaid)BIN lookup toolDetermines whether to escalate to issuer directly (via SoftSwiss) or check acquirer routing
3Check all decline codes for the failing BIN — are they all the same code, or mixed? A single code suggests one root cause; mixed codes suggest routing complexitySoftSwiss ReportsCode 05 = issuer refusing; Code 14 = bad card data; Code 91 = issuer unreachable (technical outage); Code 57 = not permitted for this merchant type
4Check if other BINs from the same issuing bank are also failing — widen BIN prefix search to all BINs from that bankSoftSwiss PortalIf all BINs from one bank are failing, it is likely a bank-wide outage or bank-level restriction on gambling merchants
5Check if the same BIN is failing on all payment methods or only one — e.g. card failing but same bank's e-wallet (Neteller) workingSoftSwiss PortalIf e-wallet from same bank works, the issue is card-scheme specific rather than bank-wide
6Check SoftSwiss routing configuration for this BIN — which acquirer is currently handling this BIN range? Is there a backup acquirer available?SoftSwiss AMThe brand cannot see routing config directly — AM must provide this. Critical for determining rerouting feasibility.
7Check whether failure is time-of-day correlated — compare decline rate for this BIN against same time yesterday and same time last weekSoftSwiss Reports + manual checkTime-correlated failure may indicate a maintenance window or bank-specific processing hours issue
8Check customer support for any player contacts from the same bank — CS agent should ask "which bank is your card with?" for all deposit failure contacts during the investigationCS DashboardConfirms real-world player impact and helps estimate volume of affected players beyond those who attempted transactions
9Check if any recent SoftSwiss configuration changes were made to this BIN or acquirer — new routing rules or fraud filters applied in the last 24 hours could cause sudden BIN failuresSoftSwiss AMIf SoftSwiss made a recent change, they can revert it; this is a fast resolution path if confirmed
10Document all findings in the incident log before contacting SoftSwiss — a structured brief will get faster resolution than an unstructured callIncident RegisterEnsures nothing is missed in escalation and provides evidence for post-incident review

SoftSwiss Escalation — BIN Specific
🔑
What to Send SoftSwiss

When raising a BIN failure ticket, always include: (1) the BIN prefix (first 6–8 digits), (2) the issuing bank and country, (3) the decline rate and volume, (4) the dominant decline code(s), (5) whether the issue is isolated to this BIN or affecting others from the same bank, (6) timestamp of first detection, and (7) your requested action (investigate / reroute / remove fraud block). The more structured the brief, the faster the resolution.
